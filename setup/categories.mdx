## What Is a Category?

A category is a single, broad classification for a post. Every post has at most **one** category (it's optional, but choosing one is encouraged). Think of categories as the top-level shelf in a library — a coarse grouping that helps people scan.

Categories are **organization-wide**: the same pool is reused across every feedback and roadmap board.

## Default Categories

When you create a workspace, ProductBridge seeds four default categories:

| Category | Use For |
|----------|---------|
| **Idea** | Feature requests and new ideas |
| **Problem** | Bug reports and complaints |
| **Question** | Things customers want clarified or explained |
| **Praise** | Positive feedback and kudos |

You can edit, remove, or add to these at any time.

## Manage Categories

<Steps>
  <Step title="Open Categories" icon="folder">
    Go to **Settings → Categories**.
  </Step>
  <Step title="Add or edit a category" icon="plus">
    Each category has:

    - **Name** — what users see (e.g., "Bug Report")
    - **Description** *(optional)* — a one-line hint shown in the picker
    - **Color** *(optional)* — used as a small swatch in admin filters and on the portal
    - **Show on Feedback** — toggle on to make this category available on feedback boards (default: on)
    - **Show on Roadmap** — toggle on to make this category available on roadmap boards (default: on)
  </Step>
  <Step title="Reorder" icon="move-vertical">
    Drag categories to change the order they appear in the category picker on the public portal.
  </Step>
  <Step title="Delete" icon="trash">
    Deleting a category doesn't delete the posts assigned to it — those posts simply lose their category. You'll be asked to confirm before deletion.
  </Step>
</Steps>

## Categories vs. Tags vs. Custom Fields

These three concepts overlap, so here's when to use which:

| Use | When | Examples |
|-----|------|----------|
| **Category** | One broad theme per post | Idea, Bug, Question, Praise |
| **Tag** | Multiple labels per post for filtering | High Impact, Mobile, Enterprise, Beta |
| **Custom Field** | Structured per-post data | Customer ARR, Urgency level, Account ID |

Rule of thumb: **categories help navigate**, **tags help filter**, **custom fields help report**.

## Hiding Categories From Specific Features

If a category only makes sense for feedback (e.g., "Bug Report") but not for roadmap items, turn off **Show on Roadmap** for that category. It will disappear from the roadmap category picker but stay available for feedback posts.

This is also useful for tracking categories — you might keep an "Internal Only" category visible to admins but hidden from the public portal.

<Callout kind="tip">
  Less is more with categories. If you find yourself with more than 8–10 categories, you're probably trying to do filtering work that [tags](/setup/tags) handle better. Keep categories broad.
</Callout>
