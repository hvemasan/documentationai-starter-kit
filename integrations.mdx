## Overview

ProductBridge supports seamless integrations with popular third-party services, version control systems, notification platforms, and custom APIs. These connections enable you to sync documentation across repositories, receive real-time alerts on changes, and automate workflows using webhooks.

<Callout kind="info">

</Callout>

## Version Control System Syncing

Set up two-way syncing with Git providers to keep your documentation in sync with code changes.

<Steps>
  <Step title="Connect Repository" icon="git-branch">
    Navigate to `Integrations > VCS` in your dashboard.
    
    Select GitHub or GitLab and authorize access.
  </Step>
  <Step title="Configure Sync" icon="settings">
    Choose branches to watch (e.g., `main`, `develop`).
    
````yaml
sync:
  branch: main
  path: docs/
  auto_publish: true
````
  </Step>
  <Step title="Test Sync" icon="play">
    Push a change to your repo and verify it appears in ProductBridge.
  </Step>
</Steps>

## API and Webhook Configurations

Use ProductBridge APIs for custom integrations and receive webhooks for real-time events.

### API Authentication

<ParamField header="Authorization" param-type="string" required="true">
  Bearer `{YOUR_API_KEY}` obtained from dashboard.
</ParamField>

<ParamField query="version" param-type="string" required="false">
  API version, defaults to `v1`.
</ParamField>

### Webhook Setup

Create webhooks for events like `doc.published`.

<Request tabs="cURL,JavaScript" show-lines="true">
```bash
curl -X POST https://api.example.com/webhooks \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-webhook-url.com/productbridge",
    "events": ["doc.published", "doc.updated"]
  }'
```

````javascript
const response = await fetch('https://api.example.com/webhooks', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    url: 'https://your-webhook-url.com/productbridge',
    events: ['doc.published', 'doc.updated']
  })
});
````
</Request>

### Sample Webhook Payload

<Response tabs="Success,Error">
```json
{
  "event": "doc.published",
  "data": {
    "id": "doc_123",
    "title": "API Guide",
    "url": "https://docs.example.com/api-guide"
  },
  "timestamp": "2024-10-15T10:30:00Z"
}
```

```json
{
  "error": "Invalid signature",
  "code": 401
}
```
</Response>

<Expandable title="Advanced Webhook Security" default-open="false">
Verify signatures using HMAC-SHA256 with your webhook secret.
  
````javascript
const crypto = require('crypto');
const signature = 'sha256=' + crypto.createHmac('sha256', secret).update(payload).digest('hex');
````
</Expandable>
