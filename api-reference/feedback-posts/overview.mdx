---
title: "Feedback posts"
description: "Individual pieces of feedback created by end-users or by your team. The core resource of the API."
---

A feedback post represents a single user request, bug report, or improvement suggestion. Posts are the primary unit of data in ProductBridge — most of the dashboard, the public portal, and the integrations are built around them.

The API lets you retrieve, list, update, merge, and soft-delete feedback posts. **Creating posts via the public API is not yet supported in v1** — posts are created via the dashboard, the public portal, integrations (Zendesk / Freshdesk sidebars), or the AI ingestion pipeline.

<Callout kind="info">
  **List filters in v1.** The `list` endpoint supports `board_id`, `status_id`, and `search` filters. Tag/author filters and Canny-style sort options (`newest` / `score` / `trending`) are not yet available — sorting defaults to newest-first.
</Callout>

## The feedback post object

<ResponseField name="id" field-type="string" required="true" deprecated="false">
  A unique identifier for the post (UUID).
</ResponseField>

<ResponseField name="title" field-type="string" required="true" deprecated="false">
  The post's title (1–512 chars).
</ResponseField>

<ResponseField name="description" field-type="string" required="false" deprecated="false">
  The post's body / description. May be `null`.
</ResponseField>

<ResponseField name="slug" field-type="string" required="true" deprecated="false">
  URL-safe slug derived from the title.
</ResponseField>

<ResponseField name="board_id" field-type="string" required="true" deprecated="false">
  UUID of the [feedback board](/api-reference/feedback-boards/overview) this post lives on.
</ResponseField>

<ResponseField name="author_id" field-type="string" required="true" deprecated="false">
  UUID of the user who created the post.
</ResponseField>

<ResponseField name="owner_id" field-type="string" required="false" deprecated="false">
  UUID of the team member who owns / is assigned to this post. May be `null`.
</ResponseField>

<ResponseField name="status_id" field-type="string" required="false" deprecated="false">
  UUID of the [status](/api-reference/statuses/overview) this post is in. May be `null` for newly-created posts before a status is assigned.
</ResponseField>

<ResponseField name="category_id" field-type="string" required="false" deprecated="false">
  UUID of the [category](/api-reference/categories/overview) this post is assigned to. May be `null`.
</ResponseField>

<ResponseField name="eta" field-type="string" required="false" deprecated="false">
  ISO 8601 date (YYYY-MM-DD) when the post is expected to ship. May be `null`.
</ResponseField>

<ResponseField name="moderation_status_code" field-type="string" required="true" deprecated="false">
  Moderation state. Common values: `"accepted"`, `"pending"`, `"rejected"`.
</ResponseField>

<ResponseField name="created_at" field-type="string" required="true" deprecated="false">
  ISO 8601 timestamp at which the post was created.
</ResponseField>

<ResponseField name="updated_at" field-type="string" required="true" deprecated="false">
  ISO 8601 timestamp of the most recent update.
</ResponseField>

## Example feedback post object

```json
{
  "id": "aa3c3ef8-b8cd-cd15-01ba-postpost0001",
  "title": "Add dark mode to the dashboard",
  "description": "It would be great to have a dark theme for late-night sessions.",
  "slug": "add-dark-mode-to-the-dashboard",
  "board_id": "553c3ef8-b8cd-cd15-01ba-12341234abcd",
  "author_id": "bb3c3ef8-b8cd-cd15-01ba-useruser0001",
  "owner_id": null,
  "status_id": "883c3ef8-b8cd-cd15-01ba-status000001",
  "category_id": "553c3ef8-b8cd-cd15-01ba-12bb12bb12bb",
  "eta": null,
  "moderation_status_code": "accepted",
  "created_at": "2026-04-15T10:30:00.000Z",
  "updated_at": "2026-04-20T12:05:11.000Z"
}
```
