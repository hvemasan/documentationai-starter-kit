---
title: "List changelog entries"
description: "Paginated list of changelog entries with optional type and label filters. Uses skip/limit instead of cursor."
---

Returns a paginated list of changelog entries for the authenticated organization. Filterable by entry `type` and by one or more `label_ids`.

<Callout kind="info">
  This endpoint accepts `skip` and `limit` (Canny-compatible offsets) on input but returns the standard cursor-paginated envelope on output. This is the only public API endpoint that uses skip-based input — all others use cursor.
</Callout>

## Endpoint

```
POST https://api.productbridge.io/api/external/v1/changelog/list
```

## Arguments

<ParamField body="api_key" param-type="string" required="true" deprecated="false">
  Your organization's public API key. See [Authentication](/api-reference/authentication).
</ParamField>

<ParamField body="type" param-type="string" required="false" deprecated="false">
  Filter to entries with this single classification. One of `"fixed"`, `"new"`, `"improved"`.
</ParamField>

<ParamField body="label_ids" param-type="array of strings" required="false" deprecated="false">
  Filter to entries that have at least one of these [labels](/api-reference/labels/overview). All ids must belong to your organization.
</ParamField>

<ParamField body="sort" param-type="string" required="false" deprecated="false">
  Sort order. One of `"newest"` (default), `"oldest"`, `"published"`.
</ParamField>

<ParamField body="limit" param-type="integer" required="false" deprecated="false">
  Number of items per page. Min 1, max 100. Defaults to 25.
</ParamField>

<ParamField body="skip" param-type="integer" required="false" deprecated="false">
  Number of items to skip from the start of the result set. Defaults to 0. Use this for offset-based pagination.
</ParamField>

## Returns

The standard cursor-paginated envelope.

```json
{
  "items": [ /* changelog entry objects */ ],
  "has_next_page": true,
  "cursor": "MjU"
}
```

## Errors

| Status | Body | Cause |
|---|---|---|
| `401` | `{"detail":{"error":"invalid api_key"}}` | Missing / unknown / inactive `api_key`. |
| `422` | Validation error envelope | `type` not in `{"fixed","new","improved"}`, `limit` out of range, malformed UUIDs in `label_ids`. |

See [Errors](/api-reference/errors) and [Pagination](/api-reference/pagination).
