---
title: "List labels"
description: "Return active changelog labels for the authenticated organization, paginated with limit + cursor."
---

Returns active changelog labels for the authenticated organization. Unlike most metadata endpoints (categories, tags, statuses), this endpoint **is paginated** because labels can grow large in active changelogs.

## Endpoint

```
POST https://api.productbridge.io/api/external/v1/labels/list
```

## Arguments

<ParamField body="api_key" param-type="string" required="true" deprecated="false">
  Your organization's public API key. See [Authentication](/api-reference/authentication).
</ParamField>

<ParamField body="limit" param-type="integer" required="false" deprecated="false">
  Number of items per page. Min 1, max 100. Defaults to 25.
</ParamField>

<ParamField body="cursor" param-type="string" required="false" deprecated="false">
  Opaque pagination cursor returned by the previous page. Omit for the first page.
</ParamField>

## Returns

The standard cursor-paginated envelope. See [Pagination](/api-reference/pagination) for the loop pattern.

```json
{
  "items": [ /* label objects */ ],
  "has_next_page": true,
  "cursor": "MjU"
}
```

## Errors

| Status | Body | Cause |
|---|---|---|
| `401` | `{"detail":{"error":"invalid api_key"}}` | Missing / unknown / inactive `api_key`. |
| `422` | Validation error envelope | `limit` out of range (must be 1–100). |
