---
title: "List comments"
description: "Paginated list of active comments for a single feedback post, in chronological order."
---

Returns a paginated list of comments attached to a single feedback post, in chronological (oldest-first) order. Replies and top-level comments are returned in the same flat array — reconstruct the tree client-side using `parent_comment_id`.

## Endpoint

```
POST https://api.productbridge.io/api/external/v1/comments/list
```

## Arguments

<ParamField body="api_key" param-type="string" required="true" deprecated="false">
  Your organization's public API key. See [Authentication](/api-reference/authentication).
</ParamField>

<ParamField body="feedback_post_id" param-type="string" required="true" deprecated="false">
  UUID of the feedback post whose comments you want.
</ParamField>

<ParamField body="limit" param-type="integer" required="false" deprecated="false">
  Number of items per page. Min 1, max 100. Defaults to 25.
</ParamField>

<ParamField body="cursor" param-type="string" required="false" deprecated="false">
  Opaque pagination cursor returned by the previous page. Omit for the first page.
</ParamField>

## Returns

The standard cursor-paginated envelope. See [Pagination](/api-reference/pagination).

```json
{
  "items": [ /* comment objects */ ],
  "has_next_page": false,
  "cursor": null
}
```

If the post belongs to another organization, the response is `200 OK` with an empty `items` array — never another org's comments.

## Errors

| Status | Body | Cause |
|---|---|---|
| `401` | `{"detail":{"error":"invalid api_key"}}` | Missing / unknown / inactive `api_key`. |
| `422` | Validation error envelope | Missing `feedback_post_id`, or it's not a UUID. |

See [Errors](/api-reference/errors) and [Pagination](/api-reference/pagination).
