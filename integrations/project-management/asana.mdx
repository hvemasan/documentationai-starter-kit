## Overview

The Asana integration connects ProductBridge to your Asana workspace so you can sync roadmap items with Asana tasks, link user feedback to development work, and keep both platforms in sync automatically.

## Prerequisites

<Callout kind="info" title="Before you start">
- An Asana account with admin permissions on the target workspace
- At least one Asana project where you want to sync tasks
- A ProductBridge project with roadmap items
</Callout>

## Connect Asana

<Steps>
  <Step title="Open Integration Settings" icon="settings">
    Navigate to **Settings > Integrations > Asana** in your ProductBridge dashboard and click **Connect**.
  </Step>
  <Step title="Authorize Access" icon="shield">
    Authorize ProductBridge via Asana's OAuth flow. ProductBridge requests access to tasks, projects, and webhooks in your workspace.
  </Step>
  <Step title="Select a Project" icon="folder">
    Choose the Asana project where ProductBridge should create and sync tasks. You can map multiple ProductBridge projects to different Asana projects.
  </Step>
</Steps>

## Configure Field Mapping

Map ProductBridge fields to Asana fields so synced tasks are structured correctly:

| ProductBridge Field | Asana Field | Description |
|--------------------|------------|-------------|
| Roadmap item title | Task name | The title of the synced task |
| Description | Task description | Full details including linked feedback |
| Status | Section | Maps to Asana sections (e.g., To Do, In Progress, Done) |
| Priority | Custom field or tag | Maps to an Asana custom field or tag |
| Assignee | Assignee | Optional: assign tasks to a specific team member |

<Steps>
  <Step title="Map Statuses" icon="columns">
    Map your ProductBridge roadmap statuses to Asana project sections. For example:

    - **Under Review** → "Backlog" section
    - **Planned** → "To Do" section
    - **In Progress** → "In Progress" section
    - **Completed** → "Done" section
  </Step>
  <Step title="Configure Priority Mapping" icon="arrow-up">
    If your Asana project uses a priority custom field, map ProductBridge priority levels (Critical, High, Medium, Low) to the corresponding Asana values.
  </Step>
  <Step title="Enable Sync" icon="refresh-cw">
    Toggle on bidirectional sync. Changes in either platform are reflected in the other within minutes.
  </Step>
</Steps>

<Callout kind="tip">
  Use Asana's custom fields to store the ProductBridge roadmap item URL. This gives your engineering team a direct link back to the feedback and context behind each task.
</Callout>

## How It Works

Once connected:

1. New roadmap items in ProductBridge create corresponding tasks in your Asana project
2. Status changes in Asana (e.g., moving a task to "Done") update the linked roadmap item in ProductBridge
3. Linked user feedback is included in the Asana task description so developers understand the context
4. Comments added in either platform are synced bidirectionally
5. When a roadmap item ships, users who submitted linked feedback can be notified automatically

<Callout kind="info">
  Bidirectional sync runs continuously. Changes in either platform are reflected within 2-5 minutes.
</Callout>

<Expandable title="Troubleshooting" default-open="false">
  - **Tasks not syncing** — Verify the OAuth token is still valid and the connected Asana project is accessible
  - **Wrong section mapping** — Check your status-to-section mapping in the integration settings. Section names must match exactly
  - **Duplicate tasks** — Ensure the integration is not connected to the same Asana project from multiple ProductBridge projects
  - **Missing custom fields** — Custom field mapping requires the field to exist in the Asana project before configuring the mapping
  - **Re-authorize** — If the connection stops working, revoke and reconnect from **Settings > Integrations > Asana**
</Expandable>
